Moodle IDOR 3.5.2 - 3.5.17

Техническая суть:

Использование валидной пользовательской сессии (cookie + sesskey)
Перебор идентификаторов пользователей (ID range enumeration)
Выполнение запросов к API без корректной проверки прав доступа

Уязвимость:
Broken Access Control / IDOR
Отсутствие проверки, принадлежит ли запрашиваемый ресурс текущему пользователю
Impact:

Массовое извлечение персональных данных пользователей:
ФИО
Email
Город проживания
Возможность автоматизированного сбора базы пользователей


moodle.py [-h] -u URL -c COOKIE -s SESSKEY [--start START] [--end END] [-o OUTPUT]

--start - от начала пользователей ( к примеру от 0 )
--end - от конца пользователей ( к примеру до 10 )
-o формат файла в котором будет сохранено к примеру moodle.json
